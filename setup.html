Google Workspace AI Assistant

A Claude Code + Google Workspace assistant for school district communications — powered by the gws CLI and Model Context Protocol.

View on GitHub

💡 What This Is

This project turns Claude Code into a personal work assistant connected to your Google Workspace account. Through a set of slash commands, it can pull your calendar, triage your inbox, manage tasks, draft social posts, check the weather, and run WiFi diagnostics — all from a single terminal interface.

No backend server required. Everything runs locally in Claude Code via the gws CLI and Model Context Protocol (MCP). Your credentials never leave your machine.

📋 Prerequisites

Claude Code
Requires a Claude Max subscription via claude.ai
Node.js v14+
Required for gws CLI and credential decryption
macOS
Some commands use macOS-specific tools (system_profiler, etc.)
Google Workspace
A Google Workspace account with Gmail, Calendar, and Tasks

🚀 Setup Steps

1
Clone the repository
git clone https://github.com/D93Labs/aigooglecli.git
cd aigooglecli
2
Install the gws CLI

The Google Workspace CLI is the bridge between Claude and your Google account.

npm install -g @googleworkspace/cli
ℹ️ This project was built with gws v0.5.0. Verify your version with gws --version.
3
Authenticate with Google

Run this in a regular terminal (not inside Claude Code). This will open a browser for OAuth sign-in.

gws auth login --account YOUR_EMAIL@yourdomain.com --full -s gmail,calendar,tasks

Your credentials are saved encrypted to ~/.config/gws/. They are never committed to git.

4
Copy and configure CLAUDE.md

The CLAUDE.example.md file is the assistant's brain — it defines your identity, district context, calendar IDs, and task list IDs. Copy it and fill in your values.

cp CLAUDE.example.md CLAUDE.md

Then open CLAUDE.md and replace every placeholder. See the Customization Guide below for the full list.

⚠️ CLAUDE.md is in .gitignore — it will never be pushed to GitHub. Keep your real IDs and personal info in this file safely.
5
Copy and configure .mcp.json

This tells Claude Code how to connect to the gws MCP server. Update the username path to match your macOS home directory.

cp mcp.example.json .mcp.json

Edit .mcp.json and replace YOUR_USERNAME with your macOS username (the folder name under /Users/).

6
Open the folder in Claude Code

Launch Claude Code, open this project folder, and Claude will automatically load .mcp.json and the slash commands from .claude/commands/.

✅ Test the connection by typing /morning or /today. If you see calendar and task data, you're set.

✏️ Customization Guide

Every placeholder in CLAUDE.example.md needs to be replaced with your real values. Here's the full list:

Placeholder What to Replace It With Where to Find It
YOUR_EMAIL@yourdomain.com Your Google Workspace email address Your email
Your Name Your full name —
Your Title Your job title —
YOUR_USERNAME Your macOS username (e.g., jsmith) Run whoami in Terminal
YOUR_SOCIAL_MEDIA_CALENDAR_ID Google Calendar resource ID for your social media calendar Google Calendar → Settings → click calendar → Calendar ID
YOUR_FAMILY_CALENDAR_ID Google Calendar resource ID for your personal/family calendar Google Calendar → Settings → click calendar → Calendar ID
YOUR_MY_TASKS_ID Google Tasks list ID for "My Tasks" Run: gws tasks tasklists list --params '{"maxResults":20}'
YOUR_TODAYS_TASKS_ID Google Tasks list ID for your daily tasks list Same as above — look for the list name in the output
YOUR_PROJECT_LIST_ID Tasks list ID for a project you're tracking Same gws command above
YOUR_MEETING_LIST_ID Tasks list ID for a recurring meeting Same gws command above
YOUR_LAT / YOUR_LON GPS coordinates for your city (for weather) Search "your city coordinates" on Google
School/org table Replace with your organization's locations or departments —
www.yourdomain.com Your organization's main website —

⚡ Slash Commands

All 14 commands are available once the project is open in Claude Code. Type any of them to run.

Command What It Does
/morningFull morning briefing — weather, calendar, inbox, and tasks in one shot
/todayReal-time check-in — what's happened so far, what's still ahead, what to focus on
/tomorrowPrep for tomorrow — tomorrow's calendar, unread emails, and open tasks
/calendarWhat's on your calendar today across all three calendars
/weekWalk through the full week, flag what to prep for, surface tasks due this week
/monthScan the rest of the month and flag anything that stands out
/inboxUnread emails from today, flagged for action where applicable
/replyDraft a reply to the most recent email needing a response (matches your voice)
/recapFull weekly recap — meetings attended, emails handled, work completed
/social-draftDraft a social media post for Facebook, Instagram, and X
/weatherCurrent conditions and today's hourly forecast (Open-Meteo, no API key needed)
/weather-week7-day forecast for your location
/wifiFull WiFi diagnostic — signal, noise, SNR, nearby APs, latency
/refresh-credsRefresh Google credentials if tools return 401 errors

🔧 Troubleshooting

MCP tools return 401 or "unauthorized" errors Your credentials have expired. Run /refresh-creds inside Claude Code, then restart Claude Code to reload the MCP server. If that doesn't work, re-authenticate from scratch (see below).
Need to re-authenticate from scratch Run this in a regular terminal (not Claude Code): 
gws auth login --account YOUR_EMAIL@yourdomain.com --full -s gmail,calendar,tasks
Calendar event creation fails with "invalid type: string, expected map" This is a confirmed bug in gws v0.5.0. The mcp__google-workspace__calendar_events_insert tool is broken for inserts. A Python subprocess workaround is documented in CLAUDE.md — Claude will use it automatically.
Slash commands not showing up Make sure you have the project folder open in Claude Code (not a parent folder). Claude Code loads commands from .claude/commands/ relative to the open directory.
